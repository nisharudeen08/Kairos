## 1. PROJECT ARCHITECTURE

Kairos is a completely autonomous, multi-agent engineering assistant built as a **VS Code Extension** and powered by a backend **LiteLLM Proxy**. 

### The Split Environment
Our project is split into two massive components:
1.  **The Extension (`/vscode`)**: A TypeScript-based VS Code Extension that provides a WebView panel. This is what you see inside the editor. It captures your messages, reads your file system natively, and intercepts the AI's commands.
2.  **The Proxy (`/litellm`)**: A Python/LiteLLM backend designed to rotate keys across FREE open-source models, preventing rate limits and abstracting API calls away from the extension.

---

## 2. THE UI & WORKFLOW (Cursor-Style)

We have recently migrated the UI to a **Glassmorphic, Tailwind-powered Design** that functions almost exactly like Cursor or Codex.

1.  **WebView Renderer (`chat.js` & `ChatViewProvider.ts`)**: 
    - The UI is styled with Tailwind CSS (`glass-panel`, `bg-white/5`).
    - It contains a persistent **Action Bar** (Changes, Terminal, Artifacts, Browser).
    - It seamlessly supports **Image Uploads** (sending Base64 images directly to Vision-capable models).
    - It manages **Conversation Memory** via VS Code's `workspaceState`.

2.  **The "Pending Review" System**:
    - The AI does **NOT** write files blindly.
    - Instead, the AI generates raw text using XML tool tags like `<write path="...">`.
    - `ChatViewProvider.ts` intercepts this `<write>` tag and prevents the AI from directly hitting the filesystem.
    - It spawns a green **"Review Code"** UI Box.
    - Only when the user clicks **"Accept"** does `chat.js` send an `acceptFile` command back to the extension host to execute `fsTools.writeFile()`.

---

## 3. THE XML TOOLING SYSTEM

We abandoned the old nested JSON execution schema in favor of a clean, strict XML schema. The orchestrator inside the VS Code backend parses the stream for three core tools:

-   `<execute>command</execute>`: Spawns a background terminal script.
-   `<read>path</read>`: Safely reads the user's workspace files.
-   `<write path="path">content</write>`: Triggers the Review Box for the user to approve a file change.
